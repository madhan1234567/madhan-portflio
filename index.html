01
Mobile UX · Service Design
EventEase — Vendor Booking App
Redesigning fragmented event coordination into a single, calm mobile experience for organisers and vendors alike
Problem Statement

Event organisers had no single place to compare vendors, confirm bookings, or track payments. They juggled WhatsApp, spreadsheets, and phone calls simultaneously — leading to double bookings and missed follow-ups.

Target Users
  • Individual event organisers (age 22–40)
  • Small business event coordinators
  • Vendors seeking reliable inbound bookings
Pain Points
  • No centralised vendor comparison by price or availability
  • Booking confirmations scattered across multiple channels
  • Payment tracking was entirely manual and error-prone
My Approach
  • Conducted user interviews with organisers and vendors
  • Built dual-sided journey maps and personas for both user types
  • Card sorting to define the information architecture
  • Lo-fi flows in Miro → Hi-fi screens in Figma → Power BI analytics simulation
Key UX Decisions
  • Single onboarding with role toggle — one app, two experiences
  • Vendor cards with embedded availability calendar
  • Integrated chat tied directly to each booking record
  • Transaction timeline view — not just a payment total
Tools Used
  • Figma (Lo-fi & Hi-fi)
  • Miro (Journey Mapping)
  • Power BI (Analytics simulation)
  • Notion (Documentation)
🎯
Prototype testing with 6 users showed an estimated 30% reduction in coordination effort per event. Vendors reported the unified request inbox removed the need for a separate CRM. Smooth dual-sided onboarding significantly reduced drop-off in simulated usability sessions.
02
AI Product · Healthcare UX
AI Skin Disease Detection — Clinical UX
Translating a deep learning model into a trustworthy diagnostic tool — bridging the gap between model performance and clinical usability
Problem Statement

A MobileNetV2 model could classify skin conditions with strong accuracy, but the raw output — a label and a confidence score — wasn't actionable for patients or GPs without dermatology expertise.

Target Users
  • Patients in lower-access areas without dermatologist access
  • General practitioners needing quick pre-screening tools
  • Healthcare data teams validating model outputs
Pain Points
  • Raw confidence percentages create anxiety without context
  • No guidance on what to do after a high-risk classification
  • Annotation pipeline lacked a structured review interface
My Approach
  • Built end-to-end data-flow diagrams (AS-IS / TO-BE) as the design foundation
  • Defined UAT acceptance criteria for each model output state
  • Mapped the annotation pipeline into a structured labelling interface
  • Applied design thinking across every stage of model validation
Key UX Decisions
  • Three-tier severity output (monitor / consult / urgent) replacing raw %-scores
  • Contextual next-step CTA on every result screen
  • Annotator dashboard with progress tracker and disagreement flags
  • Audit log for QA teams reviewing flagged predictions
Tools Used
  • Python · TensorFlow · MobileNetV2
  • Figma (Interface design)
  • Process diagrams (AS-IS/TO-BE)
  • UAT documentation
✦
Replacing confidence percentages with actionable severity tiers significantly improved result interpretability for non-technical users. The structured labelling interface reduced annotation ambiguity across multiple validation rounds and passed UAT requirements.
03
Dashboard Design · Data UX
KPI Reporting Dashboard
Turning fragmented business data from three departments into a single executive decision surface — where the right number is always visible at the right moment
Problem Statement

Three business units maintained separate reporting formats. Executives needed a unified view, but the existing layouts buried the insights that drove real decisions. Consolidating data was only half the challenge.

Target Users
  • Senior operations managers needing daily snapshots
  • Business unit leads tracking team-level KPIs
  • Executives reviewing revenue and performance trends
Pain Points
  • No single source of truth across three reporting systems
  • Key revenue signals buried inside dense data tables
  • Manual report generation consumed significant analyst time
My Approach
  • Ran structured requirements sessions with all 3 business units
  • Validated all data inputs against BI specifications before visual design
  • Applied visual hierarchy: primary KPIs at top, drill-downs beneath
  • Designed for two audiences — ops daily scan + executive weekly review
Key Design Decisions
  • Metric cards showing the delta trend, not just the raw value
  • Colour coding for SLA breach risk — not just data state
  • Customer segmentation layer revealing hidden revenue patterns
  • Automated refresh pipelines removing manual data entry
Tools Used
  • Power BI & Tableau (Dashboard design)
  • SQL (Data querying & validation)
  • Python · Pandas (Data processing)
  • Excel (Pivot tables & modelling)
📊
Dashboards signed off by all three business units and moved into production. The segmentation analysis surfaced a 15% revenue opportunity previously invisible in old reports. Manual reporting effort dropped by 30%, freeing analyst time for interpretation over collection.
